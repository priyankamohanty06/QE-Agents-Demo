Workflow Steps

  1. Test Planning: Invoke the LLM to convert the PRD/API spec into a risk-based test plan with prioritized scenarios and coverage areas, then apply a HITL checkpoint at POST_PLANNING.
  2. Test Generation: Turn the plan/specs into concrete, executable tests in a real framework, plus test data. Favor meaningful negative and boundary cases and maintainability over sheer volume.
  3. Test Execution: Execute generated tests in a safe simulated environment and collect pass/fail evidence.
  4. Defect Triaging: Invoke the LLM to turn failures into actionable defects: real vs. flaky, severity/priority, dedup and clustering, likely root cause, and owner, then apply a HITL checkpoint at POST_TRIAGE.
